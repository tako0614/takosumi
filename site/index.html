Takosumi

Self-hostable PaaS substrate. JSON-LD manifest deploys on any compute target.

What it is

Takosumi is a manifest-driven deploy engine and account plane. You write a JSON-LD Manifest describing what resources you want; the kernel resolves the graph, picks provider plugins, and applies on Cloudflare Workers, AWS, GCP, Kubernetes, or self-hosted Docker — all from the same contract.

Identity, billing, and AppInstallation ownership live in a replaceable account plane (Takosumi Accounts). Provider credentials live in a runtime-agent the operator owns. The kernel itself stays substrate-neutral and stateless.

Try it

Install the CLI and deploy a manifest against a local kernel:

deno install -gA -n takosumi jsr:@takos/takosumi-cli
takosumi server &
takosumi deploy ./manifest.yml

See the Quickstart for the full path, including how to point at Cloudflare Workers or a self-hosted Postgres + MinIO + Caddy stack.
