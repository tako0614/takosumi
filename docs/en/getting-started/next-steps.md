# Next Steps — Connect Components and HTTP Exposure {#next-steps}

After creating an Installation in the [Quickstart](./quickstart.md), continue here to add more components and expose your app.

## Connect Components {#connect-components}

After the first install succeeds, you can add components like a database and connect them to your worker.

```yaml
apiVersion: v1
metadata:
  id: com.example.hello
  name: Hello Takosumi
components:
  db:
    kind: postgres
    spec:
      version: "16"
      size: small

  web:
    kind: worker
    connect:
      db:
        output: db.connection
        inject: secret-env
        prefix: DB
    spec:
      entrypoint: src/worker.ts
```

When this manifest is deployed, the following environment variables are automatically injected into the web worker process:

```
DB_HOST=<postgres host>
DB_PORT=5432
DB_USER=<auto-generated username>
DB_PASSWORD=<auto-generated password>
```

In your worker code, read them with `Deno.env.get("DB_HOST")` or any standard environment variable API.

Apply the update to the existing Installation:

```bash
takosumi deploy inst_... --source "$APP_ROOT"
```

## Add HTTP Exposure {#add-http-exposure}

The gateway connects to the web component's HTTP output and exposes a public URL:

```
web (http output) --> gateway (connect) --> https://your-app.example.com
```

In an operator environment that provides a `gateway`, add the following to your manifest. See [HTTP Exposure](../reference/http-exposure.md) for the full specification.

```yaml
components:
  web:
    kind: worker
    spec:
      entrypoint: src/worker.ts

  public:
    kind: gateway
    connect:
      app:
        output: web.http
        inject: upstream
    spec:
      listeners:
        public:
          protocol: https
          tls: auto
      routes:
        - listener: public
          path: /
          to: app
```

Omitting `host` lets the operator assign a public hostname automatically. In local development you can set it explicitly, e.g. `host: hello.takosumi.test`. Root `publish` is not needed for browser reachability.

## Update And Roll Back {#update-and-roll-back}

Change the manifest or runtime source, then apply another Deployment:

```bash
takosumi deploy inst_... --source "$APP_ROOT"
```

A rollback moves the current pointer to a previous successful Deployment:

```bash
takosumi rollback inst_... dep_...
```
